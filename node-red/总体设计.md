# Node-RED 服务管理系统设计文档

## 1. 服务标识
**Service ID**: `node-red`

## 2. 脚本功能概览

| 脚本名称 | 主要功能 | 描述 |
|---------|---------|------|
| `install.sh` | 安装 Node-RED 服务 | 在 proot Ubuntu 环境中安装 Node.js、pnpm、下载 Node-RED、配置运行环境 |
| `start.sh` | 启动服务 | 通过 isgservicemonitor 控制文件启动 Node-RED 服务，同时启用服务自启动，移除 down 文件 rm -f "/data/data/com.termux/files/usr/var/service/node-red/down" |
| `stop.sh` | 停止服务 | 停止 Node-RED 服务，创建 `.disabled` 标志文件。同时关闭服务自启动，创建 down 文件 touch "/data/data/com.termux/files/usr/var/service/node-red/down" |
| `status.sh` | 状态查询 | 通过进程 PID 和端口 1880 检查服务状态和 HTTP 服务可用性 |
| `backup.sh` | 配置备份 | 备份 `/root/.node-red` 目录到 `/sdcard/isgbackup/node-red/` |
| `restore.sh` | 配置还原 | 还原备份文件或自动生成默认流配置文件 |
| `update.sh` | 服务升级 | 升级 Node-RED 到最新版本 |
| `uninstall.sh` | 卸载服务 | 完全卸载 Node-RED 环境和配置 |
| `autocheck.sh` | 自检监控 | 单服务自检、性能监控和健康检查 |

## 4. 服务控制命令

### 4.1 isgservicemonitor 控制
- **启动服务**: `echo u > /data/data/com.termux/files/usr/var/service/node-red/supervise/control`
- **停止服务**: `echo d > /data/data/com.termux/files/usr/var/service/node-red/supervise/control`
- **禁用自启动**: `touch /data/data/com.termux/files/usr/var/service/node-red/down`
- **启用自启动**: `rm -f /data/data/com.termux/files/usr/var/service/node-red/down`

### 4.2 进程检测机制
- 通过 1880 端口检测: `netstat -tnlp | grep 1880`
- 工作目录验证: 检查 `/proc/PID/cwd` 是否指向 node-red 相关路径
- HTTP 服务状态: 使用 `nc -z localhost 1880` 验证 Web 界面可用性

## 5. 安装部署
首先从serviceupdate.yaml读取安装需要的依赖，安装好所有依赖

### 5.1 系统环境准备
```bash
# 登录 proot-distro ubuntu 容器
proot-distro login ubuntu
```

### 5.2 Node.js 环境安装
```bash
# 安装 Node.js 20.x 和相关工具
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs npm

# 安装 pnpm 包管理器
npm install -g pnpm
```

### 5.3 Node-RED 安装
```bash
# 全局安装 Node-RED
pnpm add -g node-red@4.0.9

# 验证安装版本
cat ~/.pnpm-global/global/5/node_modules/node-red/package.json | grep '"version"' | sed -E 's/.*"version": *"([^"]+)".*/\1/'
```

### 5.4 服务运行脚本创建
```bash
# 创建服务目录
mkdir -p $PREFIX/var/service/node-red/supervise
touch $PREFIX/var/service/node-red/down

# 创建运行脚本
cat > $PREFIX/var/service/node-red/run << 'EOF'
#!/data/data/com.termux/files/usr/bin/sh
exec proot-distro login ubuntu -- bash -c "
export NODE_RED_HOME=/root/.node-red
mkdir -p \$NODE_RED_HOME
cd \$NODE_RED_HOME
exec /root/.pnpm-global/bin/node-red
" 2>&1
EOF

chmod +x $PREFIX/var/service/node-red/run
```

### 5.5 环境变量控制
```bash
# 安装指定版本
TARGET_VERSION="4.0.9" bash install.sh

# 使用默认版本安装（推荐）
bash install.sh
```

## 6. 升级维护

### 6.1 版本升级流程
首先从serviceupdate.yaml读取安装需要的依赖，安装好所有依赖

```bash
# 进入容器
proot-distro login ubuntu

# 升级 Node-RED 到指定版本

# 验证新版本
cat ~/.pnpm-global/global/5/node_modules/node-red/package.json | grep '"version"' | sed -E 's/.*"version": *"([^"]+)".*/\1/'
```

```bash

# 升级时安装指定依赖
EXTRA_DEPS="node-red-contrib-dashboard@3.1.0" bash update.sh

# 同时指定版本和依赖
TARGET_VERSION="4.0.10" EXTRA_DEPS="node-red-contrib-dashboard@3.1.0" bash update.sh
```

### 6.3 配置管理

## 7. 特殊功能

### 7.1 Web 界面访问
- **访问地址**: `http://localhost:1880`
- **管理界面**: Node-RED 流编辑器
- **API 端点**: RESTful API 和 WebSocket 连接
- **状态检查**: HTTP 连接测试验证服务可用性

### 7.2 备份策略
- **自动清理**: 保留最近 3 个备份文件
- **备份条件**: 仅在服务运行时执行备份
- **格式支持**: tar.gz 和 zip 格式
- **智能转换**: ZIP 文件自动转换为标准 tar.gz 格式

### 7.3 还原控制
```bash
# 自动还原最新备份文件
bash restore.sh

# 指定特定备份文件还原
RESTORE_FILE="/sdcard/backup/my_node_red_flows.tar.gz" bash restore.sh

# 指定 ZIP 格式备份文件（自动转换）
RESTORE_FILE="/sdcard/backup/node_red_backup.zip" bash restore.sh

# 使用外部备份文件
RESTORE_FILE="/storage/emulated/0/Download/flows-backup.zip" bash restore.sh
```

### 7.4 错误处理
- **服务依赖**: 备份前检查服务状态
- **端口占用**: 检测前确认端口可用性
- **配置验证**: 生成配置前验证 Node.js 环境
- **状态上报**: 完整的 MQTT 状态上报机制

## 8. 监控告警

## 10. 故障排查

### 10.1 常见问题
- **服务无法启动**: 检查 Node.js 环境和端口占用
- **Web界面无法访问**: 验证 1880 端口和防火墙设置
- **流执行失败**: 检查节点依赖和配置错误
- **配置文件损坏**: 使用 restore.sh 重新生成配置

### 10.2 日志查看
```bash
# 查看服务状态
bash status.sh

# 查看详细日志
tail -f /data/data/com.termux/files/home/servicemanager/node-red/logs/autocheck.log

# 查看升级步骤日志
ls /data/data/com.termux/files/home/servicemanager/node-red/logs/update_step_*.log

# 查看服务运行日志（容器内）
proot-distro login ubuntu
# Node-RED 日志输出到 isgservicemonitor 管理的日志中
```

### 10.3 手动操作
```bash
# 手动启动服务
bash start.sh

# 强制重启
bash stop.sh && sleep 5 && bash start.sh

# 重新生成配置
bash restore.sh

# 手动访问 Node-RED（容器内）
proot-distro login ubuntu

/root/.pnpm-global/bin/node-red
```

## 12. 历史记录文件

### 12.1 文件位置
- **安装历史**: `/sdcard/isgbackup/node-red/.install_history`
- **更新历史**: `/sdcard/isgbackup/node-red/.update_history`

### 12.2 记录格式

#### 安装历史记录
```
2025-07-15 15:30:15 INSTALL SUCCESS 4.0.9
2025-07-15 16:45:22 UNINSTALL SUCCESS
2025-07-16 09:15:30 INSTALL SUCCESS 4.0.10
```

#### 更新历史记录
```
2025-07-15 17:30:15 SUCCESS 4.0.8 -> 4.0.9
2025-07-15 18:45:22 FAILED 4.0.9 -> 4.0.10 (service start timeout)
2025-07-16 10:20:10 SUCCESS 4.0.9 -> 4.0.10
```

### 12.3 记录规则
- **时间格式**: `YYYY-MM-DD HH:MM:SS`
- **状态类型**: `INSTALL SUCCESS` / `UNINSTALL SUCCESS` / `SUCCESS` / `FAILED`
- **版本信息**: 安装记录单版本，更新记录版本变迁
- **失败原因**: 更新失败时包含具体原因
- **文件权限**: 存储在 `/sdcard` 目录，持久化保存

## 14. 与 Zigbee2MQTT 的差异对比

| 特性 | Node-RED | Zigbee2MQTT |
|------|----------|-------------|
| **服务类型** | HTTP Web应用 | MQTT网关服务 |
| **主要端口** | 1880 (HTTP) | 8080 (Web), MQTT |
| **配置复杂度** | 低（可视化） | 中（需串口检测） |
| **状态检查** | HTTP + 进程 | MQTT桥接 + 进程 |
| **数据格式** | JSON流配置 | YAML配置 |
| **用户界面** | 流编辑器 | 管理界面 |
| **扩展性** | 节点包生态 | Zigbee设备支持 |
| **备份内容** | 流+设置 | 配置+数据库 |
| **依赖检测** | Node.js环境 | 串口适配器 |
